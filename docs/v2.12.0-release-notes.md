# amq-squad v2.12.0

Usability and discoverability release for project-team orchestration.

## Highlights

- Added `amq-squad goal start` as the intent-first entry point for previewing
  and delivering a goal to the resolved visible lead.
- Added `amq-squad goal apply` for executing an operator-approved visible-lead
  goal after verifying a real `APPROVED:` answer on the matching gate thread.
- Added `amq-squad next` to surface the highest-priority operator action from
  gates, inbox backlog, directives, stale polling, or idle state.
- Fixed operator gate detection so a later status update on `gate/<topic>` does
  not hide an unanswered approval question from `operator status`, `notify`, or
  `next`.
- Standardized canonical action-object fields across status, operator, next,
  repair, and mutation follow-up JSON surfaces.
- Added `amq-squad operator answer` and `amq-squad operator directive` so gates
  and lead steering no longer require raw AMQ commands.
- Added repair-first topology and launch fault objects with copyable remedy
  actions.
- Expanded `doctor` with version-skew checks for the running binary, PATH
  binary, Codex skill cache, installed skill marker, AMQ, and team profile
  health.
- Added dogfood regression tests for the v2.11 runtime invariants.
- Polished goal/operator help and completion surfaces for the new commands.
- Added operator-facing documentation:
  - `docs/v2.12.0-plan.md`
  - `docs/action-object-contract.md`
  - `docs/operator-cookbook.md`
  - `docs/v2.12.0-handoff.md`

- `make readme-html-check`

- `./amq-squad version`

The stamped local binary prints `amq-squad v2.12.0`. Before install/cache
refresh, `doctor` is expected to warn when the PATH binary or Codex plugin cache
still points at v2.11.0.

## Upgrade Notes

amq-squad v2.12.0 continues to require AMQ 0.38.0 or newer.

After installing the v2.12.0 binary, refresh the Codex or Claude plugin cache so
new agent sessions load the matching `amq-squad skill v2.12.0` bundle.

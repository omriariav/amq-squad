# amq-squad v2.12.0 Handoff Digest

Generated for the v2.12.0 workstream so a fresh team can continue release
validation without reconstructing state from AMQ history.

## Release Theme

v2.12.0 is the usability-and-discoverability release. The release makes the
correct orchestration path easier to find and safer to execute: intent-first
goal startup, canonical action objects, clearer help, repair-first faults,
operator CLI paths, cookbook guidance, and fresh-team continuation context.

## Operator And Gate Rules

- No merge, release, destructive action, external send, or goal apply is
  authorized by prose alone.
- Gate approval evidence must be a real AMQ answer on `gate/<topic>` with an
  `APPROVED:` subject from the configured operator handle.
- `amq-squad operator answer` is the preferred gate answer path.
- `amq-squad operator directive` sends steering data; it does not clear gates.
- `amq-squad next` is read-only and recommends the highest-priority operator
  action for the scoped profile/session.

## Residual Risks

- `goal apply` uses the visible lead's native goal binding as the approved plan
  source for this release slice. A richer goal-plan store can be added later if
  approved plans need more structure than the native goal command.
- `goal start` currently requires explicit `--yes` for mutation in the CLI
  path. A later interactive confirmation prompt can improve ergonomics without
  changing the safety contract.
- `next` covers gate answer, operator backlog, unacknowledged directives, stale
  poll loop, and idle. More domain-specific actions can be added additively.
- The operator cookbook is documentation only; there is no docs linter in the
  current repo.

# amq-squad v2.13.0 Handoff

## Summary

v2.13.0 focused on robustness for named-profile orchestrated runs, blocked
native-goal visibility, and a smaller skill/operator surface for drafting
orchestrator invocations.

## Landed In This Slice

- Added `docs/v2.13.0-plan.md` as the milestone execution record.
- Fixed named-profile `agent up` root resolution: `--team-profile PROFILE
  --session SESSION` now derives `.agent-mail/PROFILE/SESSION` even when the
  caller omits `--root`, so AMQ does not recreate `.agent-mail/SESSION`.
- Kept explicit `--root` behavior for generated team launch commands.
- Changed legacy-root conflict detection to ignore presence-only residue such
  as `agents/<handle>/presence.json`, while still blocking real durable state.
- Added `native_goal_blocked` support in `status --json.goal_binding.mode`.
- Wired blocked native goals into `operator.poll.open_blockers`.
- Hardened status release-readiness checks for externally registered leads by
  rechecking the exact tmux pane at visibility evaluation time.
- Included issue #280 in release scope: `lead register` starts or repairs the
  external lead wake sidecar by default, supports explicit `--wake`, `--no-wake`,
  `--no-require-wake`, and wake injector flags, and persists wake metadata on
  the external launch record.
- Fixed issue #281: `resume --exec` now adopts the live tmux pane with the
  matching `amq:<session>:<role>` title after a launch-record verification
  timeout, then re-runs verification before returning a stale-record error.
- Fixed issue #279: `goal start` / `goal deliver` now accept
  `--register-orchestrator[=HANDLE]` to add/register the current non-agent pane
  as the external `orchestrator` member and start wake before goal delivery.
- Added `goal draft --skill-invocation` and `goal_draft.skill_invocation` JSON
  output for ready-to-paste `/amq-squad-orchestrator` blocks.
- Added canonical skill generation from `plugins/skills-src/*/SKILL.md` and
  `scripts/generate-plugin-skills.py`, with `make skills-check` enforcing that
  Claude and Codex plugin mirrors stay generated.
- Collapsed the primary skill surface from four skills to two:
  `amq-squad` now covers setup, role authoring, and live coordination, while
  `amq-squad-orchestrator` remains the lead-agent bootstrap skill. The former
  `amq-team-setup` and `amq-squad-role-creator` entries remain as redirect
  stubs.
- Updated Claude and Codex skill docs with blocked-goal polling guidance, the
  new skill-invocation draft path, and the two-primary-skill actor model.

## Release Risks

- `native_goal_blocked` is surfaced from launch-record goal binding metadata.
  The native goal runtime or adapter still needs to write that mode into the
  lead launch record when a goal blocks.
- Manual dogfood should still verify a fresh named-profile run dispatches
  without deleting `.agent-mail/<session>`.

## Recommended Next Actions

1. Run a fresh named-profile dogfood launch with `--team-profile` and

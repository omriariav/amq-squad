# amq-squad v2.13.0

Robustness and skill-simplification release for named-profile orchestrated
runs.

## Highlights

- Fixed named-profile `agent up` root resolution so `--team-profile PROFILE`
  with `--session SESSION` derives `.agent-mail/PROFILE/SESSION` when no
  explicit `--root` is provided.
- Kept generated team launch commands root-explicit for named profiles,
  preventing AMQ bootstrap from recreating legacy `.agent-mail/SESSION` roots.
- Relaxed legacy-root conflict detection for presence-only bootstrap residue
  such as `agents/<handle>/presence.json`, while still blocking real durable
  state.
- Added `native_goal_blocked` visibility through `status --json` goal binding
  metadata.
- Counted blocked native goals in `operator.poll.open_blockers` so stalled
  orchestrated runs surface in operator polling.
- Hardened release-readiness visibility for externally registered leads by
  rechecking their exact tmux pane before reporting a dead visible lead.
- Included `lead register` wake-sidecar support for externally registered
  orchestrator panes: wake starts by default, can be made explicit with
  `--wake`, can be skipped with `--no-wake`, can be downgraded with
  `--no-require-wake`, and supports wake injector flags.
- Fixed `resume --exec` stale-record verification by adopting the live tmux pane
  with the matching `amq:<session>:<role>` title when the agent bootstraps
  slower than the launch-record timeout.
- Added `goal start` / `goal deliver --register-orchestrator[=HANDLE]` so a
  non-agent/global-orchestrator launcher pane can be added as the
  `orchestrator` member, externally registered, and wake-enabled before goal
  delivery. The default handle is `orchestrator`.
- Added `goal draft --skill-invocation` and the matching JSON field for
  ready-to-paste `/amq-squad-orchestrator` invocations.
- Added canonical skill generation from `plugins/skills-src/*/SKILL.md` so
  Claude and Codex plugin variants share one source body and receive
  per-binary frontmatter at build/check time.
- Collapsed the primary skill surface from four skills to two:
  `amq-squad` now covers setup, role authoring, and live coordination, while
  `amq-squad-orchestrator` remains the lead-agent bootstrap skill. The former
  `amq-team-setup` and `amq-squad-role-creator` entries remain as redirect
  stubs.
- Updated Claude and Codex skill docs with blocked-goal polling guidance, the
  new skill-invocation draft path, and the two-primary-skill actor model.
- Added release planning and handoff documentation:
  - `docs/v2.13.0-plan.md`
  - `docs/v2.13.0-handoff.md`

## Upgrade Notes

amq-squad v2.13.0 continues to require AMQ 0.38.0 or newer.

After installing the v2.13.0 binary, refresh the Codex or Claude plugin cache so
new agent sessions load the matching `amq-squad skill v2.13.0` bundle.

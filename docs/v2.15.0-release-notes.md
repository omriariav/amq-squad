# amq-squad v2.15.0

Release discipline and project-lead guardrail release for milestone 2.15.0.
This release tightens the local release workflow, raises the AMQ floor to the
wake-inject fix level, and documents current model/binary guidance for
operator-driven squads.

## Highlights

- Added best-effort Claude pane renaming on spawn: Claude-launched agents receive
  `/rename <role>-<session>` after startup without blocking launch or applying to
  Codex agents (#307).
- Documented the release merge discipline in generated team rules: each PR needs
  exact-head evidence, independent review, merge preflight, and an explicit
  operator gate before merge (#308).
- Added monitor coverage for the durable-message-undrained plus idle-agent
  condition so pane-nudge or wake-delivery failures surface as operator-needed
  validation evidence instead of staying silent (#309). The timeboxed
  investigation did not land a pane-nudge corruption root-cause fix; residual
  risk is accepted for this release because detection now pulls the operator
  back to recover rather than letting the stalled handoff remain silent.
- Raised the then-current minimum supported AMQ version across doctor, docs,
  and launch guidance. This pulled in the AMQ wake-inject stale-process fix and
  kept wake-inject launches fail-fast when the sidecar could not start (#310).
- Added model, binary, and effort-selection guidance to team rules and skills,
  including point-in-time caveats and operator-setup-dependent cost framing
  (#312).
- Documented Claude model alias pass-through and explicit Codex/Claude examples
  for `up`, `agent up`, `resume`, and per-member native arguments (#313).
- Hardened project-lead adoption boundaries: a global orchestrator pane cannot
  become the project lead, project-lead auto-registration requires matching
  `AM_ROOT` proof, project-lead no-wake adoption requires a compat reason, and
  `amq-squad amq send` guards send-as for team roles unless an audited unsafe
  override is used (#314).
- Fixed named namespace collision detection so a profile and session may share
  the same slug without being mistaken for legacy default-session durable state,
  while preserving conflicts for real `.agent-mail/<session>/agents/...` state
  (#316).

## Upgrade Notes

Current amq-squad v2.16.0 builds require AMQ 0.40.0 or newer. This historical
v2.15.0 note described the prior floor raise that pulled in the AMQ
wake-inject stale-process fix.

After installing the v2.15.0 binary, refresh the Codex or Claude plugin cache so
new agent sessions load the matching `amq-squad skill v2.15.0` bundle.

Release operators should keep using the explicit gate sequence: merge, push,
tag, GitHub release, issue closure/deferral, and artifact publish each require
their own operator-approved action.

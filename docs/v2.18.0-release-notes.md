# amq-squad v2.18.0

v2.18.0 closes all 22 issues in the milestone. It hardens goal-driven squads,
makes orchestration safer and easier to start, and expands runtime support
beyond tmux without overstating what each terminal backend can do.

## Highlights

- **Bound action verification and planner-only leads.** `verify action` checks
  that a high-risk action and exact target have a matching operator approval,
  independent of the lead's trust posture. It is an explicit callable preflight,
  not shell, Git, or GitHub CLI interception (#349). Planner/reviewer-only leads
  can also be structurally barred from direct implementation (#350).
- **Safer goal delivery and namespace recovery.** `run start` waits for the lead
  to become ready before delivering its goal (#351), rejects profile/session
  namespace collisions before launch (#352), and offers non-destructive
  recovery when ambiguous durable state already exists (#353). Single-line
  pane sends also settle before submission so cold or busy TUI composers do not
  strand wake nudges (#363).
- **Smoother `run start` UX.** The built-in catalog adds persona-light lead and
  agent roles (#345), sibling tabs are the default visibility (#346), existing
  profiles fail early with actionable session-pin guidance (#347), and common
  scoped verbs accept consistent `-p`, `-s`, and `-P` aliases (#348).
  `run start --external-lead` can bind the current pane as project lead while
  spawning only the remaining workers (#340).
- **External-orchestrator evidence without task-store ambiguity.** Status and
  console expose source-labeled, read-only evidence (#335); Codex launches can
  opt into passive Symphony lifecycle telemetry (#336); and the orchestrator
  skill documents a human-gated path for adopting external evidence into a
  native task with provenance (#337).
- **Capability-aware native terminal support.** The runtime-control model keeps
  tmux at Tier A, adds iTerm2 as Tier B (#331), and adds Terminal.app as Tier C
  (#332). Native launch handling now works across non-POSIX default shells,
  records stable Terminal.app identity, and prevents terminal metadata from
  leaking into nested launches (#384-#386). cmux remains descoped pending the
  explicit re-entry bar tracked in #330.
- **Current model and documentation guidance.** Skills and examples adopt the
  gpt-5.6 Codex model family (#380), and the README was rebuilt around execution
  modes, runtime tiers, orchestration protocol, and the safety model (#388).
- **AMQ 0.41.0+ baseline.** The release adopts AMQ's wake-injection fix, reply
  receipts, and absolute-root pinning while closing the remaining wrapper-side
  wake residue (#365).

## Follow-ups

The native-terminal capability model deliberately leaves unsafe or unproven
operations disabled. Follow-up work tracks cmux control-plane feasibility
(#373), safe iTerm2 send/capture/busy detection (#374), Terminal.app
Accessibility input (#375), native-terminal close support (#376), legacy tmux
status migration (#377), cross-platform GUI backends (#378), and
capability-aware local-input detection outside tmux (#379).

Operator delivery and authority also remain explicit follow-ups: #390 tracks
pushing gate events to a human attention surface, and #391 tracks an opt-in
self-operator mode where a lead may hold delegated gate approval authority.

## Compatibility

- amq-squad v2.18.0 requires amq 0.41.0+.
- Go 1.25+ remains required.
- tmux remains the Tier A backend. iTerm2 Tier B and Terminal.app Tier C are
  macOS-only and degrade according to their advertised capabilities.
- The plugin manifests, source skill, generated Claude/Codex mirrors, and
  pinned install example are versioned together at v2.18.0.

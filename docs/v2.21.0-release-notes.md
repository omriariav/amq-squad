# amq-squad v2.21.0

## Summary

v2.21.0 contains the behavior integrated after v2.20.1. It adds trusted
authorization handoff, terminal operator-gate lifecycle, safe lead-pane wait
posture, durable task/send state, transactional namespace and orchestrator
recovery, and stricter runtime ownership checks.

## Authorization, gates, and attention

- **#414 — trusted authorization envelopes.** `verify action` can normalize a
  human-approved typed gate into an immutable Ed25519-signed authorization
  envelope. `verify authorization` requires an explicit trust store and exact
  action/target, verifies signature and revocation state, and revalidates the
  current namespace generation, durable gate and receipt bytes, and
  policy/preflight evidence before returning PASS. Compound publication keeps
  separate tag and GitHub-release child attempts, each bound to the exact active
  generation and its durable claim and receipt identity. Release eligibility
  revalidates that the tag child reached exact published evidence before the
  GitHub-release child proceeds. Durable reconciliation can adopt exact evidence
  without unsafe resend, but does not retro-claim pre-existing or otherwise
  unclaimed messages; uncertain invocation remains ambiguous, and conflicting
  evidence terminalizes the affected attempt. The verifier never executes the
  action. Approval does not activate publication, no tag or release executes
  automatically, envelopes are not bearer tokens, and the boundary does not
  intercept shell, Git, or GitHub commands.
- **#464 — terminal gate and task attention lifecycle.** `gate close` lets the
  verified requester close or withdraw only the exact current operator-gate
  generation. Terminal evidence is bound to the current request, exact raw
  thread and operator-only recipient, typed context, and AMQ reply reference;
  malformed, conflicting, degraded, stale, or non-exact binding evidence fails
  open. Same-ID copies whose on-disk recipient ordering differs are conflicting
  evidence. Answered, closed, and withdrawn gates stop notification and backlog
  escalation, while a later valid request reopens with a fresh age. Completed
  tasks project to closed attention; cancelled tasks project to closed, and a
  cancelled task with `ReplacedBy` projects to superseded. Failed and blocked
  tasks remain attention-bearing; there is no persisted superseded task state.
- **#457 — truthful operator answer routing and audits.** Operator answers and
  self-approval paths re-resolve the admitted namespace and verified actor,
  bind the current typed request, and persist durable send/audit evidence
  without claiming delivery that was not verified.

## Wait posture and durable execution

- **#416 — safe waits in the lead's own pane.** In `lead_pane` mode,
  amq-squad verifies the live lead actor before an owned blocking wait. It
  refuses waits when caller-raised operator gates remain unresolved, the
  timeout exceeds 120 seconds, or the wait is unbounded. A deliberate
  exception requires `--override-wait-posture` plus a reason and writes a
  durable audit. The guard covers `collect`, wrapped AMQ watch/receipt waits,
  and amq-squad-owned send, reply, and dispatch receipt waits; it does not
  intercept a directly invoked external `amq` or shell polling loop.
- **#461 — atomic task lifecycle.** Task completion, dependent readiness,
  optional successor claim, and delivery intents commit through a durable
  transaction journal before AMQ delivery. Claims have renewable leases, and
  `task reconcile` reports stale leases and delivery state without silently
  unclaiming work or automatically retrying uncertain delivery.
- **#462 — durable delivery receipts.** amq-squad-owned AMQ mutations reserve a
  crash-visible receipt before invocation, retain stable message IDs exposed on
  partial failures, and distinguish delivered, drained, failed, and ambiguous
  outcomes. `receipt show` refreshes one scoped receipt using the recorded
  recipient identity.
- **#463 — canonical context resolution.** Context-bearing commands select one
  canonical project/profile/session/root/actor tuple. Only mutation paths that
  acquire admission re-resolve the tuple under admission and refuse tuple drift;
  this does not imply identical mechanics for every mutation path.

## Recovery, orchestration, and runtime safety

- **#359 — stopped namespace migration.** `namespace migrate`, `recover`, and
  `rollback` provide cold-only, journaled namespace moves with read-only
  preflight, collision/device/space/schema/digest checks, and explicit recovery
  or rollback rather than best-effort partial moves.
- **#456 — transactional external orchestrator registration.** External
  orchestrator mailbox and runtime identity progress through persisted
  lifecycle states. AMQ initialization is receipt-backed, exact mailbox/runtime
  verification is required, and uncertain invocation requires explicit repair.
- **#460 — Codex goal delivery contract.** Goal delivery is claim-once and
  bound to verified launch identity. Resume preserves delivery evidence and
  requires explicit `--redeliver-goal` to create one new revalidated attempt.
- **#465 — fail-closed managed pane cleanup.** Stop/removal cleanup verifies
  pane ownership from durable manifests and normalized executable identity;
  uncertain or changed ownership is reported instead of killing or closing an
  unverified pane.
- **#467 — marker-safe managed wake injection.** Managed wake delivery strips
  bracketed-paste marker leakage instead of injecting `[200~`/`[201~` into
  agent panes.
- **#470 — fresh AMQ bootstrap.** A live start from a genuinely empty project
  initializes the selected default or named-profile AMQ root before launch,
  without falling back into a different namespace.
- **#437 — canonical workspace cleanup guidance.** Managed team rules now state
  that cleanup must be scoped, ownership-aware, and non-destructive to
  unrelated or operator-owned workspace state.

## Compatibility evidence

- **#471** adds the real-AMQ 0.43.1 compatibility harness and coverage.
- **v2.21.0 release QA** covers the supported floor/current/latest matrix,
  including canonical context and durable send/reply and receipt behavior. Its
  terminal `gate close` reply/reference proof is version-gated to AMQ 0.43.1 or
  newer.
- Go 1.25+ remains required.
- The minimum AMQ floor remains 0.42.1. The release is explicitly validated
  against pinned 0.43.1, while latest remains a forward-compatibility canary.
- The Claude and Codex plugin manifests, canonical source skill, generated
  mirrors, README install example, and startup marker are versioned together at
  v2.21.0.

## Manual Operator Test Plan

Use `/Users/omri.a/bin/amq2` directly. Do not release or install over the stable
`amq-squad` binary during this test.

Read-only baseline:

```sh
amq2 version
amq2 doctor --all-profiles
amq2 noc --root ~/Code --actions --filter project:amq-squad --json
amq2 noc --root ~/Code --filter project:amq-squad --run-action amq_env
amq2 noc --root ~/Code --filter project:amq-squad --run-action team_profiles
amq2 noc --root ~/Code --filter session:v1-0-0-qa --actions --action threads,brief,presence,amq_ops --json
```

Confirm-gated dry-run checks:

```sh
amq2 noc --root ~/Code --filter project:amq-squad --run-action new_session --set session=rc-audit --dry-run --json
amq2 noc --root ~/Code --filter project:amq-squad --run-action delete_team --set profile=default --dry-run --json
```

Interactive TUI checks:

```sh
amq2 noc --root ~/Code
```

Expected TUI behavior:

- Down arrow keeps moving the selected row and scrolls long lists.
- `p` opens the palette and finds `new session`, `delete team`, `team rules`,
  `team profiles`, `amq env`, `amq who`, and session actions.
- Mutating actions open a preview/confirm flow before execution.
- Read-only actions show output without opening a destructive confirm.
- `q` exits cleanly.

Optional disposable mutation smoke:

```sh
tmp="$(mktemp -d /tmp/amq2-operator-smoke.XXXXXX)"
git -C "$tmp" init
amq2 noc --root "$tmp" --actions --action new_team --json
amq2 noc --root "$tmp" --run-action new_team --set roles=cto,qa --set binary=cto=sleep,qa=sleep --set session=operator-smoke --yes
amq2 noc --root "$tmp" --actions --action new_session,delete_team,team_profiles --json
amq2 noc --root "$tmp" --run-action delete_team --set profile=default --dry-run --json
```

Pass criteria:

- `doctor --all-profiles` has no hard failures for AMQ version, team config,
  tmux, or pointer sync.
- NOC action JSON exposes creation, session, deletion, profile, rules, AMQ env,
  AMQ who, AMQ ops, thread, brief, and presence paths where applicable.
- `new_session` dry-run reports `would_execute=false`, `requires_confirmation=true`,
  valid template values, and passing preflight checks.
- `delete_team` dry-run reports `would_execute=false`, `requires_confirmation=true`,
  and leaves sessions, briefs, rules, and pointer stubs untouched.
- Real TUI navigation and overlays feel coherent enough for a live operator.
